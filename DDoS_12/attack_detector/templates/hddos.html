Understanding DoS & DDoS Attacks

Explore various types of attack vectors and how they impact networks.

DDoS from Random IPs

Involves flooding the target with traffic from thousands of spoofed or infected IP addresses, making it difficult to block malicious sources and overwhelming the network or server.

SYN Flood

Exploits the TCP handshake by sending numerous SYN requests without completing connections, overwhelming the target system.

RST Attack

Sends forged TCP reset (RST) packets to terminate active connections, disrupting legitimate traffic flow.

ACK Flood

Overloads networks by sending large numbers of TCP ACK packets, bypassing some firewalls and targeting bandwidth.

FIN Flood

Bombards the server with TCP FIN packets, aiming to exhaust resources by tricking the system into closing connections.

PSH Flood

Uses TCP PSH flags to force the server to process packets immediately, creating high CPU usage and service disruption.

UDP Flood

Sends massive volumes of UDP packets to random ports, causing the target to repeatedly check for nonexistent applications.

Fragmentation (FRAG) Attack

Breaks packets into smaller pieces to bypass security measures, potentially crashing target systems or consuming resources.

ICMP Flood

Overwhelms the target with ICMP Echo Request (ping) packets, often used in smurf attacks to cause network congestion.

Benign Traffic

Legitimate user activity such as web browsing, streaming, or downloading — used as a baseline to detect anomalies.

Comprehensive Explanation of DoS and DDoS Attacks

Denial of Service (DoS) is a type of cyberattack where a single machine or attacker attempts to overwhelm a service, server, or network by flooding it with excessive traffic or triggering resource exhaustion through specially crafted requests. The goal is to disrupt normal functionality and deny access to legitimate users.

Distributed Denial of Service (DDoS) extends the concept by launching the attack from multiple devices—often part of a botnet (a network of infected devices controlled by the attacker). DDoS attacks are more difficult to detect and mitigate due to their distributed nature and massive scale.

Types of DoS/DDoS Attacks

DDoS and DoS attacks can be categorized into several types based on which part of the system they target:

  • Volumetric Attacks: These flood the network with massive amounts of traffic to saturate bandwidth.
    Examples: UDP Flood, ICMP Flood, DNS Amplification
  • Protocol Attacks: Exploit weaknesses in Layer 3 (network) and Layer 4 (transport) protocols to exhaust server resources or state tables in networking equipment.
    Examples: SYN Flood, Ping of Death, Smurf Attack, Fragmentation Attack
  • Application-Layer Attacks: Mimic legitimate user behavior to exhaust application resources such as CPU, memory, or database connections.
    Examples: HTTP Flood, Slowloris, DNS Query Flood

Attack Techniques

  • Botnets: Networks of compromised devices used to launch DDoS attacks globally.
  • Reflection & Amplification: Use vulnerable servers (e.g., DNS, NTP) to amplify traffic sent to the target.
  • Spoofing: Attackers forge IP addresses to hide the source and complicate filtering.

Real-World Impact

  • Website Downtime: E-commerce, banks, and news outlets can suffer financial and reputational loss.
  • Infrastructure Damage: Extended attacks can cause service instability or hardware stress.
  • Security Diversion: DDoS attacks are sometimes used as distractions while attackers infiltrate other systems.

Mitigation Strategies

  • Rate Limiting: Restrict the number of requests from a user or IP in a given timeframe.
  • Web Application Firewalls (WAF): Filter and monitor HTTP traffic to prevent app-layer attacks.
  • Geo-Blocking & IP Blacklisting: Block traffic from known malicious IP ranges.
  • Anycast Routing: Distribute traffic across global servers to absorb large-scale attacks.
  • Third-party DDoS Protection Services: Services like Cloudflare, AWS Shield, or Akamai offer real-time detection and mitigation.

Understanding and mitigating DoS and DDoS attacks is essential to maintaining service availability and user trust in today’s internet-dependent world.

Mitigation Strategies for DoS & DDoS

Explore advanced techniques to identify and block attacks effectively.

1. Traffic Filtering & Rate Limiting

What it does: Filters out malicious traffic patterns before reaching critical services.

How:

  • Use iptables, firewalls, or IPS to block suspicious/spoofed IPs.
  • Limit packets per second (PPS) per IP.
  • Drop malformed packets or protocol anomalies.

2. Machine Learning-Based Anomaly Detection

What it does: Detects unusual traffic behavior using trained ML models.

How:

  • Train ML models (e.g., Random Forest, XGBoost) on traffic features.
  • Use PyShark or Scapy to stream real-time data into the model.
  • Trigger alerts or blocks on abnormal spikes/repetitions.

3. IP Blacklisting with Auto-Block

What it does: Blocks attacker IPs automatically after detection.

How:

  • Store blocked IPs in Django's database.
  • Use Linux iptables/ebtables to enforce blocking.
  • Allow manual unblock/investigation from UI.

4. Rate-Based Load Balancing

What it does: Distributes traffic to reduce overload risk.

How:

  • Set thresholds for CPU, memory, and packet rate.
  • Redirect excessive load to dummy servers or delay queues.
  • Use load balancers like HAProxy or Nginx.

5. Honeypot Diversion & Trap Systems

What it does: Diverts suspicious traffic to decoy environments.

How:

  • Deploy low-interaction honeypots simulating vulnerable services.
  • Route flagged traffic to honeypot for behavior analysis.
  • Use findings to enhance blacklist and mitigation rules.
